### ¿Qué significa "comparar diferencias entre documentos"?

Generalmente, se refiere a identificar y resaltar:
*   **Adiciones:** Texto, imágenes o elementos que están en un documento pero no en el otro.
*   **Eliminaciones:** Texto, imágenes o elementos que están en un documento pero fueron removidos en la versión comparada.
*   **Modificaciones:** Cambios en el texto (palabras, frases), formato (fuente, tamaño, color), o estructura (tablas, imágenes movidas/redimensionadas).

### Herramientas y Métodos para Comparar Documentos:

#### 1. Para Documentos de Texto (Microsoft Word, Google Docs, etc.)

*   **Microsoft Word (Función "Comparar")**:
    *   **Cómo funciona:** Word tiene una función integrada (en la pestaña "Revisar" > "Comparar") que te permite seleccionar dos documentos de Word (original y revisado) y genera un tercer documento que muestra todas las diferencias con "marcas de revisión" (texto añadido subrayado, texto eliminado tachado, cambios de formato). Puedes ver el documento original, el revisado y el documento de comparación al mismo tiempo.
    *   **Ideal para:** Contratos, informes, artículos, cualquier documento colaborativo o que ha pasado por varias revisiones y necesitas ver los cambios exactos.
    *   **Ventajas:** Muy preciso para documentos de Word, resalta tanto texto como formato.
    *   **Desventajas:** Solo funciona con archivos .doc o .docx.

*   **Google Docs (Historial de Versiones)**:
    *   **Cómo funciona:** Google Docs guarda automáticamente todas las versiones de un documento. Puedes ir a "Archivo" > "Historial de versiones" > "Ver historial de versiones". Aquí puedes seleccionar dos versiones diferentes y Google Docs te mostrará las diferencias resaltadas.
    *   **Ideal para:** Colaboración en tiempo real, seguimiento de cambios en documentos compartidos en la nube.
    *   **Ventajas:** Integrado, fácil de usar, permite volver a versiones anteriores.
    *   **Desventajas:** No genera un documento "final" con las revisiones al estilo Word, es más una vista del historial.

*   **LibreOffice Writer (Comparar Documentos)**:
    *   **Cómo funciona:** Similar a Word, ofrece una función de "Comparar documentos" para archivos ODT y otros formatos compatibles.
    *   **Ideal para:** Usuarios de suites de oficina de código abierto.

#### 2. Para Documentos PDF

*   **Adobe Acrobat Pro (Comparar Archivos)**:
    *   **Cómo funciona:** Acrobat Pro tiene una herramienta de "Comparar Archivos" que detecta rápidamente las diferencias entre dos documentos PDF. Puede comparar texto, imágenes e incluso la estructura visual de las páginas. Presenta un informe detallado de los cambios.
    *   **Ideal para:** Contratos legales, manuales técnicos, planos, o cualquier documento que se distribuye en formato PDF y requiere una revisión exhaustiva de cambios.
    *   **Ventajas:** Muy potente y preciso para PDFs, puede comparar documentos escaneados (mediante OCR).
    *   **Desventajas:** Es una herramienta de pago.

*   **Herramientas Online de Comparación de PDF (Ej: Draftable, Diffchecker PDF)**:
    *   **Cómo funciona:** Subes los dos archivos PDF a un sitio web y este te muestra las diferencias, a menudo resaltadas o lado a lado.
    *   **Ideal para:** Comparaciones rápidas de PDF sin necesidad de software instalado.
    *   **Ventajas:** Accesible desde cualquier navegador, a menudo gratuito para usos limitados.
    *   **Desventajas:** Puede tener límites de tamaño de archivo, problemas de privacidad si el documento es sensible, no siempre tan preciso como las herramientas de escritorio.

#### 3. Para Código y Texto Plano

*   **Herramientas Diff/Merge (Ej: WinMerge, Beyond Compare, KDiff3, Diffchecker)**:
    *   **Cómo funciona:** Estas herramientas son ideales para comparar archivos de texto plano (código fuente, scripts, archivos de configuración, etc.). Muestran las diferencias línea por línea, resaltando adiciones, eliminaciones y cambios. Muchas de ellas también permiten "fusionar" (merge) los cambios de un archivo al otro.
    *   **Ideal para:** Desarrolladores de software, administradores de sistemas, escritores que trabajan con Markdown o texto plano.
    *   **Ventajas:** Extremadamente rápidas y precisas para texto plano, visualización clara.
    *   **Desventajas:** No manejan formato enriquecido (negritas, cursivas), imágenes, o estructuras complejas de documentos.

*   **Integración en IDEs (Entornos de Desarrollo Integrados)**:
    *   **Cómo funciona:** Muchos IDEs (como VS Code, IntelliJ IDEA) tienen funcionalidades de comparación de archivos integradas, a menudo ligadas a sistemas de control de versiones (Git).
    *   **Ideal para:** Desarrolladores que necesitan comparar versiones de su código.

#### 4. Para Imágenes o Archivos Binarios

*   **Herramientas de Comparación de Imágenes (Ej: Diff-JPEG, herramientas de edición de imagen como Photoshop con capas)**:
    *   **Cómo funciona:** Estas herramientas superponen dos imágenes y resaltan las diferencias de píxeles o regiones. En software de edición, puedes poner las imágenes en capas y usar modos de fusión para ver los cambios.
    *   **Ideal para:** Diseñadores gráficos, control de calidad visual.
    *   **Ventajas:** Permite detectar cambios visuales sutiles.
    *   **Desventajas:** No es "comparación de documentos" en el sentido tradicional.

### Pasos Generales para Comparar Documentos:

1.  **Identifica el tipo de documento:** ¿Es un Word, un PDF, un archivo de texto plano?
2.  **Elige la herramienta adecuada:** Basado en el tipo de documento y tus necesidades.
3.  **Carga/Abre los dos documentos:** El documento "original" y el documento "revisado" o "a comparar".
4.  **Ejecuta la comparación:** La herramienta te mostrará las diferencias.
5.  **Revisa las diferencias:** Evalúa los cambios y decide si son correctos, si hay que aceptar o rechazar revisiones, o fusionar el contenido.

```assembly
    num1        DW  1234        ; Primer número (Define Word = 16 bits)

    MOV AH, 09h
    LEA DX, crlf
    INT 21h

    ; --- Finalizar el programa ---
    MOV AH, 4Ch         ; Función DOS para terminar el programa
    INT 21h             ; Llama a la interrupción DOS para salir
MAIN ENDP

; --- Subrutina para imprimir un número de 16 bits sin signo ---
; Recibe el número en AX
Print_Num PROC
    PUSH AX             ; Guarda los registros que se modificarán
    PUSH BX
    PUSH CX
    PUSH DX

    MOV CX, 0           ; CX será el contador de dígitos
    MOV BX, 10          ; El divisor para obtener los dígitos es 10

DivisionLoop:
    XOR DX, DX          ; Limpia DX (necesario para la operación DIV de 16 bits)
    DIV BX              ; Divide AX por 10.
                        ;   AX contendrá el cociente
                        ;   DX contendrá el residuo (el dígito actual)
    PUSH DX             ; Empuja el dígito (residuo) a la pila
    INC CX              ; Incrementa el contador de dígitos
    CMP AX, 0           ; Compara el cociente con 0
    JNE DivisionLoop    ; Si el cociente no es 0, sigue dividiendo

PrintLoop:
    POP DX              ; Saca un dígito de la pila
    ADD DL, '0'         ; Convierte el dígito numérico a su carácter ASCII (ej: 0 a '0')
    MOV AH, 02h         ; Función DOS para imprimir un carácter
    INT 21h             ; Llama a la interrupción DOS para mostrar el carácter
    LOOP PrintLoop      ; Decrementa CX y salta a PrintLoop si CX no es 0

    POP DX              ; Restaura los registros guardados
    POP CX
    POP BX
    POP AX
    RET                 ; Retorna de la subrutina
Print_Num ENDP

END MAIN            ; Fin del programa principal
```

**Explicación de algunas partes clave:**

*   **`.MODEL SMALL`**: Define el modelo de memoria. Para programas pequeños y simples es lo más común.
*   **`.DATA` y `.CODE`**: Separan las secciones de datos y de código.
*   **`MOV AX, @DATA` y `MOV DS, AX`**: Inicializan el registro de segmento de datos (`DS`) con la dirección del segmento de datos. Esto es crucial para que el programa pueda acceder a las variables definidas en `.DATA`.
*   **`INT 21h`**: Es la interrupción principal del DOS. Diferentes valores en el registro `AH` le dicen al DOS qué función ejecutar:
    *   `AH = 09h`: Imprime una cadena de caracteres terminada en '$'. La dirección de la cadena debe estar en `DX`.
    *   `AH = 02h`: Imprime un solo carácter. El carácter debe estar en `DL`.
    *   `AH = 4Ch`: Termina el programa y devuelve el control al DOS.
*   **`Print_Num` Subrutina**: Esta subrutina es fundamental para mostrar números en pantalla, ya que el DOS solo puede imprimir caracteres.
    *   Divide repetidamente el número por 10.
    *   Los residuos de cada división son los dígitos (en orden inverso).
    *   Se usan `PUSH` para guardar los dígitos en la pila y luego `POP` para sacarlos en el orden correcto para imprimir.
    *   `ADD DL, '0'` convierte un dígito numérico (ej. 5) a su valor ASCII ('5').
